## BAGIAN 5 — Baseline Knowledge (Quick Check)

Bagian ini bukan ujian — kalibrasi pemahaman dasar.

**24. Manakah pernyataan yang PALING tepat tentang Large Language Model (LLM)?**

- LLM menjawab dengan mencari informasi real-time dari Google
- LLM memprediksi token berikutnya berdasarkan pola dari data training
- LLM punya database fakta yang selalu up-to-date
- LLM selalu menghasilkan jawaban deterministik (sama persis tiap kali)

**25. "Context window" pada LLM merujuk pada:**

- Ukuran jendela aplikasi chat
- Jumlah token maksimum yang bisa diproses dalam satu request (input + output)
- Bahasa yang didukung
- Jumlah pengguna yang dilayani bersamaan

**26. Few-shot prompting adalah teknik dengan memberi LLM:**

- Beberapa contoh input-output di dalam prompt agar model meniru polanya
- Beberapa pertanyaan sekaligus
- Akses internet untuk mencari jawaban
- Banyak parameter tuning

**27. RAG (Retrieval Augmented Generation) pada dasarnya adalah:**

- Mencari dokumen relevan → menyisipkannya ke prompt → LLM menjawab berdasar dokumen
- Melatih ulang LLM dengan data baru
- Membuat LLM lebih cepat
- Mengurangi biaya API

**28. AI Agent berbeda dari Chatbot biasa karena Agent:**

- Selalu memakai voice
- Bisa merencanakan, memilih tool, mengeksekusi action multi-step untuk mencapai tujuan
- Hanya bisa menjawab pertanyaan
- Tidak butuh LLM

**29. "Prompt injection" adalah risiko keamanan dimana:**

- API key bocor di kode
- Attacker menyisipkan instruksi berbahaya lewat input user agar LLM mengabaikan instruksi sistem
- Server LLM down
- Token habis

**30. Praktik PALING aman saat membangun AI app dengan data sensitif:**

- Kirim seluruh database ke prompt LLM
- Hardcode API key di frontend
- Gunakan retrieval hanya untuk data yang relevan, sanitize input, simpan API key di backend, audit log
- Matikan semua filter agar respons lebih cepat
